# `/git` 与 `/read` 指令需求文档

## 1. `/git`：同步工作区

### 触发方式
用户在对话中发送 `/git`。

### 功能目标
自动完成工作区代码的提交、拉取与推送，使本地分支与远程保持同步。

### 详细需求

1. **检测本地变更**
   - 检查当前工作区是否存在未提交的变更（新增、修改、删除的文件）。

2. **自动提交**
   - 若存在未提交变更，自动执行 `git add -A` 将所有变更加入暂存区。
   - 使用固定模板生成 commit message，模板中应包含当前 UTC 时间戳，例如：
     ```
     chore: sync workspace at 2026-06-23 03:25:43 UTC
     ```
   - 执行 `git commit` 完成提交。
   - 若无本地变更，跳过提交步骤并明确告知用户。

3. **拉取远程更新**
   - 执行 `git pull`。
   - 若拉取失败（如冲突、无上游分支），应终止后续 push 并返回错误信息。

4. **推送本地提交**
   - 执行 `git push`。
   - 若推送失败，返回错误信息。

5. **结果反馈**
   - 返回执行摘要，包含每个步骤的状态（提交、拉取、推送）。
   - 若某一步失败，返回已执行步骤的摘要及失败原因。

6. **非交互式安全约束**
   - 执行 git 操作时应禁用编辑器，避免在非交互环境中挂起。

---

## 2. `/read <文件名或相对路径>`：读取 Markdown 文件

### 触发方式
用户在对话中发送 `/read <文件名>` 或 `/read <相对路径.md>`。

### 功能目标
根据用户提供的名称或相对路径，返回工作区内对应 Markdown 文件的完整内容。

### 详细需求

1. **参数校验**
   - 若用户未提供参数，返回用法提示。

2. **按文件名查找**
   - 当参数仅包含文件名（不含 `/` 或 `\` 路径分隔符）时，在工作区内递归搜索所有 `.md` 文件。
   - 匹配规则：
     - 若用户输入已包含 `.md` 后缀，按原样匹配。
     - 若用户输入未包含 `.md` 后缀，自动补全为 `<输入>.md` 进行匹配。
   - 文件名匹配不区分大小写。

3. **按相对路径读取**
   - 当参数包含路径分隔符时，将其视为相对于工作区的相对路径。
   - 仅当目标文件位于工作区内时才允许读取。
   - 仅允许读取后缀为 `.md` 的文件。

4. **多匹配处理**
   - 若按文件名搜索到多个匹配的 Markdown 文件，应列出所有候选文件的相对路径，并提示用户通过相对路径精确指定。

5. **未找到处理**
   - 若未找到任何匹配文件，返回未找到提示。

6. **输出内容**
   - 对于单个匹配文件，返回其完整文本内容。
   - 输出应以纯文本/Markdown 形式渲染。

7. **安全约束**
   - 禁止读取工作区之外的文件（路径穿越防护）。
   - 禁止读取非 Markdown 文件。

---

## 3. 通用需求

1. **作用范围**
   - 两个指令的操作范围均应限定在 nanobot 当前 agent workspace 内，不应扩展到其他任意目录。

2. **错误反馈**
   - 所有失败情况（非 Git 仓库、无权限、文件不存在、命令执行失败等）都应向用户返回清晰、可读的提示。

3. **帮助与命令面板**
   - 两个指令应出现在 `/help` 输出及内置命令面板（command palette）中，包含简洁标题、描述与参数提示。
