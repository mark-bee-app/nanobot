## 🏗️ 第二阶段：核心架构（1 天）

### 目标
理解 Agent 循环、消息总线、配置系统等核心组件。

### 核心文件阅读顺序

#### 1. 配置系统
```
nanobot/config/schema.py      # 配置数据结构定义
nanobot/config/loader.py      # 配置加载逻辑
nanobot/config/paths.py       # 路径管理（workspace, 数据目录等）
```
**理解**: 配置如何定义、加载、环境变量解析

#### 2. 消息总线
```
nanobot/bus/queue.py          # 消息队列实现
nanobot/bus/events.py         # 事件定义（InboundMessage, OutboundMessage）
```
**理解**: 消息如何在系统内传递

#### 3. Agent 核心循环（重点）
```
nanobot/agent/loop.py         # AgentLoop 类 - 核心处理引擎
nanobot/agent/runner.py       # AgentRunner - LLM 调用执行器
nanobot/agent/context.py      # ContextBuilder - 上下文构建
nanobot/agent/hook.py         # 生命周期钩子机制
```
**理解**:
- Agent 如何接收消息 → 构建上下文 → 调用 LLM → 执行工具 → 返回响应
- 钩子机制如何工作（before_iteration, on_stream, after_iteration 等）
- 会话管理如何隔离不同用户的对话历史

#### 4. 工具系统
```
nanobot/agent/tools/registry.py    # 工具注册表
nanobot/agent/tools/base.py        # 工具基类
nanobot/agent/tools/*.py           # 具体工具实现
  - filesystem.py    # 文件操作
  - shell.py         # 命令执行
  - web.py           # 网页搜索/抓取
  - search.py        # 代码搜索 (grep/glob)
  - spawn.py         # 子代理
  - mcp.py           # MCP 协议支持
  - cron.py          # 定时任务
```
**理解**: 工具如何定义、注册、被 Agent 调用

### 实践任务
- [ ] 画出 AgentLoop 的处理流程图
- [ ] 追踪一个工具从定义到被调用的完整路径
- [ ] 理解钩子机制的设计模式
